How it works

From raw back-office to decision — in seconds

A browser plugin or a simple HTTP API does the capturing. CustomerPulse does the thinking.

01

Capture anything

A point-and-click browser plugin scrapes any CRM or back-office page, or platforms push events straight to the ingestion API. Both map into the same canonical schema.

02

One canonical profile

Entity, value, activity, risk and tags — a single fixed shape. Your vertical config relabels it: ARR, GGR, AUM or LTV, all from the same core.

03

AI brief + next action

A vertical-tuned AI reads the pattern between the numbers, names the archetype, flags risk, and recommends one concrete next step for your team.

Ingestion

Two ways in. One profile.

However your data arrives, it lands in the same canonical store — org-isolated and locked down.

🧩 Browser plugin

Works on any CRM/back-office. Point-and-click mapping learns your page's labels once, then captures the full profile in a click and opens it in CustomerPulse.

🔌 HTTP ingestion API

Platforms push identity, session and transaction events with a per-org API key. Kept light — only logins and payments. Row-level-security isolates every org.
